什么是 JSX 语法

JSX 是一种类 XML 语言，是 JavaScript 的语法扩展，全称是 JavaScriptXML。实质上这只是一个语法糖，每个 XML 标签都会被 JSX 转换工具转换成纯 JavaScript 代码。

为什么要使用 JSX 语法

JSX 可以使组件的结构和组件间的关系看过去更清晰。

State （对于需要改变的数据使用 state ）

对于 state，我们需要知道 5 点。

  • 在 constructor 中初始化 state
  • 父组件的 state 发生改变，子组件会重新渲染
  • 使用 setState 修改 state，并且setState 是异步的
  • setState 是批处理的，多次调用 setState 时，它会把每次传给 setState 的对象合并在一起形成一个新的对象。
  • setState 传入的是函数时，多次调用 setState，它会按照调用的顺序把这些方法排个队，把上个状态传给当前方法。

React Native 生命周期

一些常用生命周期简介

  • render
    这个方法是必须的
    对视图进行渲染
    在render 里不能改变 state，要保持 render 的纯净。render 必须返回一个值（React element、string或是numbers、portals、booleans、null）
    返回 null 或者 false 来表明不需要渲染任何东西。

  • componentDidMount
    渲染执行之后立刻调用，如果要改变 DOM 或是 获取远程数据最好放在这里。

  • componentWillReceiveProps (nextProps)
    组件接收到新的 props 时调用，即父组件修改子组件属性时触发，可用于更新某个 state 来响应 props 的变化。

  • shouldComponentUpdate (nextProps, nextState)
    在接收到新的 props 或者 state，将要渲染之前调用,如果确定新的 props 和 state 不会导致组件更新，则此处应该 返回 false。返回true将进行渲染。

  • componentWillUnmount()
    在组件从 DOM 中移除的时候立刻被调用。在该方法中执行任何必要的清理，比如无效的定时器，或者清除在 componentDidMount 中创建的 DOM 元素。
